wteh.autoaltwhsecd - WT Billed Status
  
Label: WT Billed Status
Type: character
Format: x
Decimals: ?
Initial: 
Extent: 0
Mandatory: no
Val Exp: ?
Val Msg: 
Help: O)pen, B)illed or Blank
Trigger: 



Description:
If a WT will be used to automatically generate an OE Order - WT Billed Order,
then the field will carry a value.
During receiving - WTEI, WLEBR, or WTEE, if a WT qualifies as a transfer
for billing of customer owned inventory transfer, it flags the WT with 'O' for
Open in this field.
Once the WT is used to create the OE Order for billing of the transferred
goods, the field value goes to 'B' for Billed if the order creates clean.
If the WT contains a Non-Stock line item, the WT Billed status will be set to
'E'rror. Meaning, the Non Stock portion of the WT must be billed manually
by someone.
If the order does not meet the requirements - the field remains blank.
See oeeh.autoaltwhsefl for edits of the WT to validate as a WT Billed Order.
The WT can be set to 'B'illed as well from OEEPI when a normal sales line
contains a line with AltWhse loaded. OEEPI will auto generate a WT and
auto receive it. At that point, if set to use 'Bill and Hold' processing, then
the WT is set to Billed and the OE is tied to the WT Header as the Billed OE#.
 
Content:
B - Billed
E - Error
O - Open
" " - Blank
