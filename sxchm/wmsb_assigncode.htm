wmsb.assigncode - Assignment
  
Label: Assignment
Type: character
Format: x
Decimals: 0
Initial: o
Extent: 0
Mandatory: no
Val Exp: ?
Val Msg: 
Help: (O)pen, (P)rimary, (S)ingle, (A)lternate, (X)Staging, (U)navail
Trigger: 

Indexes:
k-wmassign k-wmstatus


Description:
Assign code
 
Content:
O = open Can be assigned or unassigned. An assigned open bin has
product assigned to be stored in it while an unassigned bin
does not have any products assigned. If you are using
random storage or FIFO picking, open unassigned bins are
used. Each bin can have its own pick priority.
P = primary The main storage location for a product. They are used for
picking and will be replenished when product is received or
when it contains less than its minimum quantity. A product
can have only one primary location.
S = single Used with products that have a single location that does not
change. A product that is stored in a single bin cannot be
stored in any other bin. Single bins, unlike primary bins, do
not have any related open or alternate bins for the same
product. Single bins are assumed to be of infinite size.
A = alternate Secondary locations most often used to replenish primary
locations. A product can have many alternate locations, each
one with its own pick priority.
X = staging Used when products are temporarily stored in a location so
you can locate and sort portions of orders during packing
and shipping. This provides visibility of containers picked by
operators and facilitates moving, locating, and retrieving the
containers to quickly consolidate an order. Staging locations
are ideal for orders with Will Call or Ship Complete
dispositions. For example, products are stored in a staging
area until back orders are filled. Staging bins allow you to
store more than one product regardless of your administrator
options. Pickers will be directed to extract products from
staging bins first.
U = unavailable
