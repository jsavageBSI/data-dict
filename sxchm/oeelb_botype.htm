oeelb.botype - B/O
  
Label: B/O

Format: x

Mandatory: no
Val Exp: ?
Val Msg: 
Help: (Y)es, (N)o, (P)ickup
Trigger: 



Description:
The back order type for the order. This field defaults to "d" on a DO
order, "y" if the customer accepts back orders, or "n" if the customer
does not accept back orders.
 
Content:
y = create a back order if necessary
n = do not create a back order for this line
d = this is a drop ship line (all lines on a DO order type have a
botype of d) (inventory is not affected)
p = this is a pick-up-on-the-way order (inventory is not affected)
