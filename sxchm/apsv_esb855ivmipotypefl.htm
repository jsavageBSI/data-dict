apsv.esb855ivmipotypefl - Create VMI Orders as Purchase Orders
  
Label: Create VMI Orders as Purchase Orders
Type: logical
Format: yes/no
Decimals: ?
Initial: no
Extent: 0
Mandatory: yes
Val Exp: ?
Val Msg: 
Help: Used as ION Default Setting for EDEPA 855In BOD Interface
Trigger: 



Description:
Used on acknowledgements that are VMI specific orders.
Database default is 'no', so will still create orders as Quote orders until set to yes.
Used in the 855in Purchase Order Acknowledgement interface (EDEPA) for ION or EDI.
It is used as a replacement of adding an option to the report for EDI processing of
EDEPA 855in. If turned on, the BOD Interface will create Purchase Orders as PO
orders, not QU orders. Allows the distributor to skip the process of validating all
the purchase order data first and then convert the Quote orders to PO orders
for processing. If the vendor is a trusted vendor and sends reliable updates
and changes to the purchase orders, then use this option.
The logic changed is also used in POEVP where outbound edi852 PO orders are
created. Option to create empty PO orders for the Vendor to use based on the
recommndeded inventory to reorder. Used in the edi852 in process (EDEPA) when
loading the empty PO orders with items.
