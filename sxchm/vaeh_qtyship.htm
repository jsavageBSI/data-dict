vaeh.qtyship - Qty Ship

Description:
The quantity that will be fabricated for the value add transaction. It
is always set to be the same as the quantity ordered, up until the time
the finished product is received. At that time, the actual qty that was
fabricated will be obtained from the user and placed in the qtyship.
This is needed to determine if a value add transaction backorder should
be created. If a value add transaction backorder is created, then the
original suffix will contain the qty ordered and the actual qty shipped
(different than qty ordered) and a new suffix will be created contain
the difference between the 2 qtys from the original suffix and stored in
both the qty ordered and qty shipped of the new suffix. The qty shipped
is represented in the banner unit of measure.
 
Content:
total qty shipped of the line item
