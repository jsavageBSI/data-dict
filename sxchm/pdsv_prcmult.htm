pdsv.prcmult - Price
  
Label: Price
Type: decimal
Format: zzzzzz9.99999
Decimals: 5
Initial: 0
Extent: 9
Mandatory: no
Val Exp: ?
Val Msg: 
Help: 
Trigger: 



Description:
Price. This field provides a price for the PO. Note that this is an
array of 9, each element representing another quantity break. If no
quantity breaks are entered the first element will be used.
Otherwise, the first level will be used until the first quantity break
is met, then the program will proceed to the second level.
Note that all prices are considered to be in stocking or costing unit
(special pricing/costing) and that all prices are in the currency for
the vendor from which the products are purchased.
Note also that this is a hard dollar cost, not a multiplier or a
percentage.
If no quantity breaks are needed or quantity breaks are to be applied
against discount levels, the 'vendorprice' field should be used and this
field (ALL ELEMENTS) must be set to '0'.
