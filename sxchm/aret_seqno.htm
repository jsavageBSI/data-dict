Description:
The Sequence Number is a numeric field with valid values from 000 to
999. It is used as key field to further indentify type 0, 3, and
11 transactions.
Type 0-Invoice records created in OEEPI have a sequence
equal to 000. Type 0-Invoice records created in ARECE through the
F8-Debit Memo function have a sequence number equal to 000.
But, type O-Invoice records created in ARET have a sequence number
equal to 001.
Type 3-Unapplied Cash transactions can be created in OEET, F6-Rcv on
Acct. In this case the sequence number is 001. If the type
3-Unapplied Cash is created in ARECE, the sequence number can go up
to 999 depending on the number of transactions.
Type 11-Payment records have a different sequence number for each
payment of an invoice.
