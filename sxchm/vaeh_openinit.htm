vaeh.openinit - Open Initials
  
Label: Open Initials
Type: character
Format: x(4)
Decimals: ?
Initial: 
Extent: 0
Mandatory: no
Val Exp: ?
Val Msg: 
Help: 
Trigger: 



Description:
The operator that is currently working on the value add transaction.
this field is used to soft lock the order. When entry or maintenance is
performed on an order, the openinit is loaded with the current operator.
When the operator closes the order, the openinit is cleared. This
prevents 2 operators from working on the same order at the same time. We
employ soft locking since we are not in a transaction state the entire
time an order is entered due to potential lock table overflow problems
and length of record lock problems.
 
Content:
g-operinit when an order is opened
