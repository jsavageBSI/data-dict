apsv.esballowinvnopofl - Allow Invoices Without PO
  
Label: Allow Invoices Without PO
Type: logical
Format: yes/no
Decimals: ?
Initial: no
Extent: 0
Mandatory: yes
Val Exp: ?
Val Msg: 
Help: 
Trigger: 



Description:
Vendor Default setting used via the ION procesSupplierInvoice interface.
If an Invoice is sent without a Purchase Order, or the data errors out with
bad data in the interface and the PO is prevented from being created in
APECE (apeid records), should the Invoice be created in the Batch?
If not, no PO, No Invoice is created in APECE. If allowed, then will still
create the Invoice.
