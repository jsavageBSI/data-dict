Label: Tied Fulfillment Order
Type: logical
Format: yes/no
Decimals: ?

Extent: 0
Mandatory: yes
Val Exp: ?
Val Msg: 
Help: 
Trigger: 



Description:
Tied Fulfillment Order are created from the original Fulfillment Order once
the operator selects the Warehouses or Ties to be used to fill the line items
and Submit is choosen from OEET, OEIF, or SXAPI Full Order Maint (based
on OESF SXAPI Auto Submit option setting).
If a Stock Order is tied to a Fulfillment Order (Created from it), then the tied
flag is set. The only tie to the fulfillment order is through the OEELO record
that is attached to the original fulfillment order - showing what Tied
Fulfillment Orders are associated with the fulfillment order. Once the Tied Order is
created, the Tied Order shows the original fulfillment order it came from
by looking at the fulfillment orderno/oordersuf fields.
