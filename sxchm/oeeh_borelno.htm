oeeh.borelno - Back Order of Release #
  
Label: Back Order of Release #
Type: integer
Format: >9
Decimals: ?
Initial: 0
Extent: 0
Mandatory: no
Val Exp: ?
Val Msg: 
Help: 
Trigger: 



Description:
If borelfl is yes (indicating this is a back order), borelno contains the release which was back ordered. This field is relevant for BR and JITS. For example: if 01 represents a blanket release, and 02 is a back order of 01, and 03 is a back order of 02, then the borelfl for both 02 and 03 will be 01. Even though 03 is a backorder from 02, the ORIGINAL release order is 01 and the borelfl will point to the ORIGINAL release.
