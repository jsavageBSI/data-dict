Description:
This field stores an additional order rep that can be assigned to an order. Setups in ARSC will copy to ARSS on a copy and can use the AO logic to copy to ARSS if ARSC is changed (SXE-2803). When an order is created, OEEH defaults from ARSC/ARSS and then OEEL defaults from OEEH. The operator can update the assignments in OEEH/OEEL for individual order/line specifics. The values must be valid in OESS (SalesRep Setup). Once the data is stored in the OEEH/OEEL records in the database, it will be sent to DataLake and any analysis/calculations (usually for commissions) against these fields will be done in Birst. There are five of these fields that can be used.
Added with JIRA SXE-3048/SXWEB-38216.
