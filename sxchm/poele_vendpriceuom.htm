poele.vendpriceuom - Vendor Price UOM
  
Label: Vendor Price UOM
Type: character
Format: x(4)
Decimals: ?
Initial: 
Extent: 0
Mandatory: no
Val Exp: ?
Val Msg: 
Help: Vendor's Price UOM, If Vendor's Price UOM &lt;&gt; Price/Cost Per
Trigger: 



Description:
If the price in the flat file is different from the POEL.PRICE in the
Purchase Order, the price unit of measure from the flat file will be
written to this field to provide the user with a possible research
method to determine what caused the price to be different. The edi unit
of measure codes come in in the flat file and are then translated to the
Trend unit of measure codes based on the data in ICSEU or SASTT. The
correlating Trend unit of measure code will be written to the file if
the cross reference can be found, otherwise, the edi unit of measure
code will be written directly from the flat file.
