Description:
Start Date. This field is required, indicating the first effective date
of this record. Start date is an element in the primary key for PDSV.
This allows two records for the same product but with different start
dates. It is not possible to have two records for the same company,
vendor, product, and startdt.
It is possible to define records with future and past start dates.
Future dated records will not become effective until the start date.
Back dated records will become effective immediately unless another
record exists with a more recent start date (and other valid
information), in which case that become effective only if that more
recent record expires, is inactivated, or is deleted.
