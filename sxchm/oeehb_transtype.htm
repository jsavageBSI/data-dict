oeehb.transtype - Type
  
Label: Type
Type: character
Format: x(2)
Decimals: ?
Initial: 
Extent: 0
Mandatory: no
Val Exp: ?
Val Msg: 
Help: SO, QU, FO, CR, or RM
Trigger: 



Description:
The transaction type of the order. This describes the kind of order it
is. Supported through batch are:
SO - stock order (reserves inventory. Entry into stage 1)
QU - quote (does not reserve inventory. Entry into stage 0. Converted/
copied through the F7 copy)
FO - future order (does not reserve inventory. Entry into stage 0.
Converted to a SO through OEEPC or OEEPI when the promise date -
oeeh.longltdays - sasc.oefutdays - sasc.oereqdays &lt;= today)
 
Content:
RM - return merchandise (does not reserve inventory. Reduces on hand
once invoiced. Entered into stage 1. Can be applied against an
open invoice for accounting purposes or an invoiced sale order for
inventory return purposes)
CR - correction (a debit or credit memo. Does not affect inventory at
all. Entered into stage 3. Can be applied against an open invoice
for accounting purposes)
The SO orders create type 0 or 4 ARETs. The RM creates a type 5 (if
applied) or 6 ARET. The CR creates a type 5 (if applied) or 6
