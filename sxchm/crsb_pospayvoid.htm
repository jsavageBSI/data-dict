crsb.pospayvoid - Positive Pay Void
  
Label: Positive Pay Void
Type: character
Format: x(1)
Decimals: ?
Initial: I
Extent: 0
Mandatory: no
Val Exp: ?
Val Msg: 
Help: (I)nclude or (E)xclude VOID Transactions in Positive Pay
Trigger: 



Description:
This character field is used to indicate whether the bank's Positive Pay
program will accept data when a check is voided.
The default is that Void information can be sent to the bank and will be
used by the Positive Pay program
 
Content:
I - Include Void information
E - Exclude Void information
