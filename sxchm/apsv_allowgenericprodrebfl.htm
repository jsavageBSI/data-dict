Description:
Added with JIRA SXWEB-38291/SXWEB-39527. This field sets this vendor to participate on the Generic Rebates logic for Vendor on Sale rebates, when checked. It is located in APSV – Ordering – Rebates section with the label “Use Generic Rebates”. When a product is entered on the OE line, the vendor is looked up – in ICSW, ICSC, or from OEEL depending on the product being a stock, catalog, or non-stock product – to determine if that vendor is a generic product rebate vendor based on the APSV allow generic rebates flag. If that vendor allows generic rebates, the PDSR rebate logic will ignore the assigned vendor, except to limit the choices to vendors who use generic rebates. As soon as a record is found, ignoring the vendor, that rebate will be used and then the logic will continue to use all other criteria for finding the best rebate to use, e.g., it will still use the Customer Rebate Types listed for the customer in ARSC/ARSS and any other criteria such as customer/shipto/warehouse. So it is just ignoring the vendor when searching for the rebate to use but does not ignore any other criteria.
Usage of this logic requires AOData PD - PDGenericVendProdRebateFl to be checked. This can be found in AO – Products – Rebates – Rebates Options section with the label “Allow Generic Product Rebates”.
