oeelk.processtatfl - Processed
  
Label: Processed
Type: logical
Format: yes/no
Decimals: ?
Initial: no
Extent: 0
Mandatory: yes
Val Exp: ?
Val Msg: 
Help: 
Trigger: 



Description:
This field is used to control the KPEA update recovery logic. It is
initially set to no. All component records that have not been processed
in KPEA will be read upon F10 update. If the machine goes down during
the update, only those components not yet processed will be read for
processing when the procedure is reactivated. It is set to yes once all
processing is performed on a component through KPEA.
 
Content:
no = not update thru KPEA
yes = update thru KPEA
