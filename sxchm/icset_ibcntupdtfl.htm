icset.ibcntupdtfl - IB Update Count
  
Label: IB Update Count
Type: logical
Format: yes/no
Decimals: ?
Initial: no
Extent: 0
Mandatory: yes
Val Exp: ?
Val Msg: 
Help: Update With Quantities Scanned Through Integrated Barcode
Trigger: 



Description:
Indicates whether or not a physical/cycle count should be updated with the quantities scanned in through Integrated Barcode.
It is set based on the ICEPR option "Prn -R)eport,(U)pd Count Flag,or (B)oth?". If you are using Integrated Barcode to capture data, this option allows the user to print the reconciliation report, update the count with the quantities scanned in or both.
 
Content:
Yes - The cycle/physical count should be updated based on the quantities scanned in through Integrated Barcode
No - The cycle/physical count should not be updated
