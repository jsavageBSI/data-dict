Indexes:
k-apinvno k-inq k-invdt k-status


Description:
Description: This field represents the desired vendor # for the
transaction you are processing. After the user enters the vendor number
the system edits this field against the APSV file, where the vendor
number is stored. If the vendor number does not exist in the APSV file
the sytem will error the user stating that the vendor number does not
exist. If a asterisk or exclamation point appears next to this field
this means that a note exists and the user may or will be required to
review this note. Vendor notes are also stored and retrieved from the
APSV file. An exclamation point is a required note.
Enterable: This field is enterable by the user and is a required field
for APET processing.
Allowable Values: This field is stored as a decimal value up to 12
integer positions in length with 0 decimal positions. This allows the
user the maximum numeric length the Progress language has to offer.
