oeeh.fulfillmentbillcd - Fulfillment Billing Code
  
Label: Fulfillment Billing Code
Type: character
Format: x(2)
Decimals: ?
Initial: 
Extent: 0
Mandatory: no
Val Exp: ?
Val Msg: 
Help: 
Trigger: 



Description:
Determines when a Fulfillment Order is ready to be billed through
Consolidated Invoicing.
When OERC is run, it looks at all the fulfillment orders submitted and looks
at the Tied Order stock status to see if the billing setting has been met.
If an order is set to Manually Release, the means someone is forcing the order
through Consolidated Invoicing and they manually set the code through
OEIF. Once an order has been validated based on the Quantity Ship status
of the tied orders, the oeeh.fulfillmentstgcd is set to 'R'eleased.
OERC will pick up the 'R'eleased and 'M'anually released fulfillment stage
orders
 
Content:
R-Manually Released - Must Manually Release from OEIF
S-Shipped Completely - Order set to Released once fully shipped
S25-Shipped 25% Complete - Bills Once 25% of the Fulfillment Order is Shipped
S50-Shipped 50% Complete - Bills Once 50% of the Fulfillment Order is Shipped
S75-Shipped 75% Complete - Bills Once 75% of the Fulfillment Order is Shipped
