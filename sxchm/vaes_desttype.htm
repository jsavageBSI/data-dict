vaes.desttype - Destination Type
  
Label: Destination Type
Type: character
Format: x
Decimals: ?
Initial: 
Extent: 0
Mandatory: no
Val Exp: ?
Val Msg: 
Help: 
Trigger: 



Description:
Most value add transaction sections define a process that involves
inventory. The destination type field is used to specify where the
inventory is to be sent after that process.
 
Content:
The destination type field can contain:
v - Another server vendor. That is, this section will involved
inventory that is to be sent to another service vendor for
additional processing.
w - An internal processing warehouse. This section is to send the
inventory to another internal processing warehouse.
f - The final processing step. The inventory is to be send to the
primary warehouse since the value add transaction is not considered
a Direct Order.
c - The final processing step could send the inventory directly to the
customer if the value add transaction is a Direct Order that is tied
to a single sales order.
