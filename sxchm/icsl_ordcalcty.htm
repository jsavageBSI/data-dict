icsl.ordcalcty - Order Method
  
Label: Order Method
Type: character
Format: x(1)
Decimals: ?
Initial: C
Extent: 0
Mandatory: no
Val Exp: ?
Val Msg: 
Help: (E)oq,(C)lass,(M)in/max,(Q)ty Break,(B)lanket Order, or (H)uman
Trigger: 



Description:
The order calculate type determines the method to be used in calculating
the quantity to order when this product needs to be ordered. It is used
by ICAM to determine and set the order quantity. Note that it is not
used by RRAR except in the case of Min/Max. Order quantities are
calculated dynamically for Min/Max. This field also appears on a large
number of IC and PO reports.
 
Content:
(E)oq,(C)lass,(M)in/max,(Q)ty Break,(B)lanket Order, or (H)uman
E-EOQ Use standard EOQ calculation method
C-Class Use Gordon Graham standard inventory class method
M-Min/Max Use Order Point, order to Line Point
Q-Qty Brk Use PDSV Quantity Break record to calculate best qty
B-Blanket Do not calculate an order quantity
H-Human Manually set, do not override
