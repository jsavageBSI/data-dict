pv_oeln.orderreppct1 - Order Rep 1 Percent
  
Label: Order Rep 1 Percent
Type: decimal

Val Exp: ?
Val Msg: 
Help:

Description:
This field stores an additional order rep percentage that can be assigned to an order. Setups in ARSC will copy to ARSS on a copy and can use the AO logic to copy to ARSS if ARSC is changed (SXE-2803). When an order is created, OEEH defaults from ARSC/ARSS and then OEEL defaults from OEEH. The operator can update the assignments in OEEH/OEEL for individual order/line specifics. Once the data is stored in the OEEH/OEEL records in the database, it will be sent to DataLake and any analysis/calculations (usually for commissions) against these fields will be done in Birst. There are five of these fields that can be used. Note that the field number must match to the ordrep field number, i.e., if ordrep2 is assigned a value and you want a percentage related to that, the percentage must be specified in orderreppct2, the 2 must match on both fields to correlate them.
Added with JIRA SXE-3048/SXWEB-38216.
