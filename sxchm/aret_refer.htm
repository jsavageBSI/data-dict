aret.refer - Ref
  
Label: Ref
Type: character
Format: x(24)
Decimals: ?
Initial: 
Extent: 0
Mandatory: no
Val Exp: ?
Val Msg: 
Help: User Defined or Reference # (Start With . or #)
Trigger: 



Description:
The Referenence is a text field. It is enterable by the user in ARET,
ARECE, AREBT, and OEET(header).
Type 3-Unapplied Cash transactions created in OEET will create an
aret.refer of "OE Received On Account".
Type 7-Check transactions created in OEET will contain an aret.refer
of the order number if a reference is not entered in the OEET header.
ARET records created in OEEPI will have an aret.refer the same as the
OEET header reference.
Type 1-Service Charges and the corresponding type 11-Payment records
created in ARER will contain "On Balance of" and the amount the
service charge was calculated on.
The type 7-Check transactions will usually contain the aret.refer "Cash Rec Jrnl" and the journal number from ARECE. If it is for a split check, the type 7-Check transactions will have an aret.refer of "SPLIT CHECK".
Type 3-Unapplied Cash transactions will contain the aret.refer "AR
Cash Rec Jrnl" and the journal number from ARECE.
Types 0-Invoice and 11-Payment transactions for CODs will create aret.refer's of "Uncoll COD Amt-DD/MM/YY" if the full amount is not paid.
Type 11-Payment Records will have an aret.refer of "Split Into #
Pmts" if the number of payments is changed in apemp.
