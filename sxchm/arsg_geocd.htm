Description:
Taxware Sales and Use Tax (SUT) uses the geo code to determine which
taxes should be applied if there is more than one taxing jurisdiction within a
zip code. Examples of this are when a zip code crosses county lines and
each county has a different tax rate or when multiple cities exist in the same
zip code and each city has a different tax rate. Each taxing jurisdiction
within the zip code is given a unique, two-digit identifier. Taxware uses the
city name, state, zip code and geo code to find the correct tax rates.
Taxware Enterprise (TWE) uses the geo code as a unique identifier of a
taxing jurisdiction and not a supplemental identifier as in SUT. With TWE,
all jusrisdictions world-wide have a geo code. Each geo code can include
country, state or province, county, city and district taxing jurisdictions. Any
taxing jurisdiction except country can be omitted from the definition of a
geo code. TWE can apply the correct tax rates based on the geo code
alone.
