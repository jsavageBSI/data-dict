oeeh.transtype - Type
  
Label: Type
Type: character
Format: x(2)
Decimals: ?
Initial: 
Extent: 0
Mandatory: yes
Val Exp: ?
Val Msg: 
Help: SO, DO, F0, ST, CS, CR, QU, BL, or RM
Trigger: 



Description:
The transaction type of the order.
 
Content:
This describes the kind of order it is. Types include:
SO - stock order (reserves inventory. Entry into stage 1)
CS - counter sale (reserves inventory. Entry into stage 3)
DO - direct order (does not reserve inventory. Entry into stage 1)
QU - quote (does not reserve inventory. Entry into stage 0. Converted/
copied through OEEPC or OEET File - Copy)
ST - standing order (does not reserve inventory. Entry into stage 0.
Converted to a SO or DO through OEEPC or OEEPI when the promise
date - sasc.oereqdays &lt;= today)
FO - future order (does not reserve inventory. Entry into stage 0.
Converted to a SO through OEEPC or OEEPI when the promise date -
oeeh.longltdays - sasc.oefutdays - sasc.oereqdays &lt;= today)
BL - blanket order (does not reserve inventory. Entry into stage 0.
Always a memo order. BR suffix created from OEERB from
the header screen)
Order Fulfillment project uses the BL order to create the initial fulfillment
product demand, so a BL can be flagged as a Fulfillment Order, no longer
using it as a traditional Blanket Order (oeeh.fulfillmentordfl = yes)
